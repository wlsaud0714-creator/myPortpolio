# myPortpolio
컴퓨터 구조 및 운영체제 수업

여기다가 정리 시작.
MD -> markdown 문법.

컴퓨터가 이해하는 정보
data -> 숫자나 문자같은 정보 
동작 방식 모음,데이터를 움직이는 코드 -> 명령어 
ex) 1과 2를 더하라 
1,2 는 data 더하라 는 명렁어

컴퓨터 부품: CPU,메모리, 보조기억장치,입출력장치
메모리: 현재 진행되는 프로그램에 명령어와 데이터를 저장하는 공간.
메모리에 저장된 값의 위치는 주소로 알 수 있다.

CPU: 메모리에 저장된 명렁어를 읽고 처리하는공간. ALU 제어장치 레지스터로 구성.
ALU: CPU안에 있고 계산을 주로 담당.
제어장치: 제어신호 주고받기 명령.
레지스터: 일시적 저장공간. 명령어 해석

보조기억장치 메모리보다 크기가 큼 저장된 내용을 잃지않음.
시스템 버스: 주소 버스, 데이터 버스, 제어 버스 
단위 (Unit)	기호	크기	쉽게 이해하기 (예시)		
비트 (Bit)	b	0 또는 1	컴퓨터가 인식하는 가장 작은 단위 (전구 스위치의 켜짐/꺼짐)		
바이트 (Byte)	B	8 Bits	영문 알파벳 1글자, 혹은 숫자 1개를 저장하는 크기		
킬로바이트 (Kilobyte)	KB	1,024 Bytes	짧은 텍스트 메모, 이메일 한 통		
메가바이트 (Megabyte)	MB	1,024 KB	고화질 스마트폰 사진 1장 (약 3~5MB), MP3 노래 1곡		
기가바이트 (Gigabyte)	GB	1,024 MB	2시간짜리 고화질 영화 1편 (약 2~4GB)		
테라바이트 (Terabyte)	TB	1,024 GB	요즘 나오는 개인용 PC 하드디스크나 외장하드의 전체 용량

-> 11 이면 0과1을 뒤집고 1을 더해도 가능하다.
